# Pixel Twins

Pixel Twinsは、移植性のあるゲームシステムライブラリおよびハードウェアランタイムです。
WizwardはPixel Twins上で動作する最初のゲームですが、Pixel Twins自体は特定のゲームに
依存せず、Wizwardへの依存も持ちません。

対象システムは、主に以下の機能で構成されます。

- 2枚のLEDパネル
- 2台のUSBゲームコントローラー
- 波形メモリ音源
- 音楽および効果音用シーケンサ
- スプライト描画システム
- ゲームに必要なタイミング制御とハードウェア抽象化

## 移植性の目標

Pixel Twinsは、macOS向けホスト実装と対象マイコンの両方をサポートします。
ゲームコード、スプライト処理、シーケンサ、フレームバッファ生成には、両環境で同じ
C++ソースコードを使用することを目指します。

基本となるプラットフォーム境界は、フレームバッファ生成の後に置きます。

```text
ゲームおよびシステムの状態
        |
        v
共通C++によるスプライト処理と描画
        |
        v
共通形式のフレームバッファ
        |
        +--> macOSではSDLで表示
        |
        +--> マイコンではLEDパネルドライバへ転送
```

macOS向けの初期実装にはSDLを使用します。SDLはホスト環境との薄い接続層とし、完成した
フレームバッファのウィンドウへの転送と、必要に応じた入力および音声出力の仲介だけを
担当します。SDLの描画機能を使ってフレームバッファ生成を別実装にしてはいけません。

将来実装するLEDパネル向けコードでは、フレームバッファ転送とハードウェア接続部分だけを
置き換えます。これにより、描画処理の挙動をマイコンで動かす前にmacOS上で検証できます。

フレームバッファの寸法、メモリ配置、パレット、予約色、スプライトの透明色については、
[フレームバッファ仕様](docs/framebuffer.md)で定義します。

BG、スプライト、拡大縮小、プリミティブ、Yバケットについては、
[描画機能仕様](docs/rendering.md)で定義します。

変換済みBGとスプライトの実機向け形式は、[BGアセットバイナリ仕様](docs/background_asset_binary.md)と
[スプライトアセットバイナリ仕様](docs/sprite_asset_binary.md)で定義します。

ゲームから見えるコントローラー入力については、
[コントローラー仕様](docs/controller.md)で定義します。

## 共通実装の方針

移植可能なC++コードを基準実装とします。プラットフォーム固有コードは、次のような
ハードウェアおよびホストI/Oとの境界に限定します。

- 完成したフレームバッファのSDLまたはLEDドライバへの転送
- USBゲームコントローラーの状態取得
- 生成した音声のホストまたは対象音声ハードウェアへの出力
- クロック、タイマー、その他の不可避なボードサポート

スプライトの挙動、ゲームから見える描画規則、シーケンサのロジック、フレームバッファ生成に、
プラットフォーム条件分岐を持ち込まないことを目指します。

性能上必要な場合は、対象環境に固有のアセンブラ最適化を許容します。ただし、共通C++実装と
同じ挙動およびデータ契約を維持し、macOS版を基準実装およびデバッグ環境として利用できる
状態を保ちます。

## 初期開発の順序

最初にSDLを使ったmacOS向けホスト実装を開発します。初期マイルストーンは以下のとおりです。

1. 共通フレームバッファ形式と表示インターフェースを定義する
2. 最小限のSDL接続層でフレームバッファを表示する
3. 共通スプライトシステムを実装し、共通フレームバッファへ描画する
4. 2台のゲームコントローラーに対応する
5. 波形メモリ音源とシーケンサを実装する
6. 同じインターフェースに対するマイコンおよびLEDパネル接続層を実装する

## macOS版のビルド

macOS版はC++17、CMake、SDL3を使用します。SDL3をインストールした環境で、次のように
ビルドおよびテストできます。

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build --parallel
ctest --test-dir build --output-on-failure
```

フレームバッファ表示サンプルは、次のコマンドで実行します。

```sh
./build/examples/framebuffer/pixel_twins_framebuffer_example
```

現在の実装には、以下が含まれます。

- 8ビットパレット式ダブルフレームバッファ
- 全画面、左画面、右画面のRenderTarget
- 2の冪サイズ・8ビットタイル番号によるBG描画とピクセル単位スクロール
- 通常スプライト描画
- 最近傍法による拡大縮小付きスプライト描画
- 固定長スプライトアロケーター
- 180本のYバケット
- 矩形、ライン、塗り三角形、円・楕円のプリミティブ描画
- 固定縁色・可変文字色・可変strideの8×9フォント描画
- 2人分の共通コントローラー状態とSDL3入力
- SDL3によるmacOS表示

BGM用シーケンスJSONは `tools/sequence_converter.py`、効果音プリセットJSONは
`tools/sfx_converter.py` で、マイコンから直接参照できるC++定数へ変換できます。
変換後の再生時にはパースや動的確保を行いません。効果音は `SfxPreset` と
`makeSfxRequest()` を使って再生要求を組み立てます。

画像群の共通パレット生成、8-bitインデックスPNGへの変換、減色品質と容量のレポート生成には、
[アセットコンバーター](tools/asset_converter/README.md)を使用します。共通パレット中間画像に加え、
パレット、BG、スプライトの実機用バイナリを生成します。
