# コントローラー仕様

## 目的

ゲームコードは、SDLやUSBホストの型を直接使用しません。対象固有の入力処理は毎フレーム
`ControllerSample`を2人分生成し、共通の`Controllers`へ渡します。ゲームは更新後の
`ControllerState`だけを参照します。

`Controllers`は動的メモリを使用せず、2人分を固定長配列で保持します。

## 状態

- `x`, `y`: 左スティックまたは方向キー。`int16_t`の全範囲を使用し、中央は0
- `held`: 現在押されているボタン
- `pressed`: このフレームで押されたボタン
- `released`: このフレームで離されたボタン
- `connected`: そのプレイヤーに利用可能な入力経路があるか
- `gamepad`: 物理ゲームパッドが割り当てられているか。操作表示の切り替えに利用できる

方向は左、上、右、下の4ビット、選択ボタンも画面上の位置に合わせた左、上、右、下の
4ビットとして表します。このため、ゲーム側はXbox系・PS系コントローラーのボタン名や
番号を意識しません。このほかに`start`と`back`を定義します。

切断時を含め、対象固有処理は毎フレームすべての値を提示します。`Controllers::update()`が
前フレームとの差から`pressed`と`released`を生成します。

## SDL実装

SDL実装は標準ゲームパッドを接続順にP1、P2へ割り当てます。左スティックには中心付近の
誤差を除くデッドゾーンを設け、十字キーは同じ`x`, `y`へも反映します。フェイスボタンは
物理的な位置で次のように変換します。

- West: 選択・左（PS系の□）
- North: 選択・上（PS系の△）
- East: 選択・右（PS系の○）
- South: 選択・下（PS系の×）

macOSでの移植とデバッグ用に、ゲームパッド入力へ次のキーボード入力を重ねます。

| 操作 | P1 | P2 |
| --- | --- | --- |
| 移動 | WASD | カーソルキー |
| 選択・左/上/右/下 | 1/2/3/4 | 7/8/9/0 |
| Start | P、Space、Esc | — |

`Presenter`、`ControllerInput`の順に生成します。イベントは
`Presenter::processEvents(&controllerInput)`へ渡し、その後に
`controllerInput.update(controllers)`を1回呼びます。入力のエッジはこの`update`単位です。
