# サウンドシステム仕様

## 目的と境界

Pixel Twinsは、波形メモリ音源、BGM・SFXシーケンサ、およびPCMミキサーを共通C++で
実装します。macOS版では、共通コードが生成したPCMをSDLへ渡すだけとします。将来の
RP2350版では、同じPCM生成処理の出力をDMA転送します。

SDLやDMAの都合を音源、エンベロープ、シーケンサの挙動へ持ち込んではいけません。

## PCM出力

- サンプルレート: 48kHz
- 形式: 符号付き16bitステレオ
- 配列形式: L、Rのインターリーブ
- 生成単位: 512サンプルフレーム
- 1ブロックの時間: 約10.67ms

共通側のPCM生成APIは、512サンプルフレームのブロックを1回の呼び出しで生成します。
RP2350版ではDMA転送完了割り込みを契機として、次に必要となるブロックを生成することを
想定します。具体的なDMA構成はRP2350接続層の実装時に決めますが、「念のため」の追加
バッファは確保しません。

## ボイス

- BGM: 固定8ボイス
- SFX: 固定8ボイス
- 合計: 固定16ボイス
- 波形テーブル: 1周期32サンプル

BGMのボイス番号は変換済みシーケンスデータが明示します。同じBGMボイス番号へ新しい
ノートが来た場合は、そのボイスを置き換えます。SFXのボイス割り当て規則と、全ボイス使用時の
優先順位はSFX APIの設計時に定義します。

実行中の動的メモリ確保は行いません。

## 数値表現

RP2350の浮動小数点演算を利用し、エンベロープ、音量、パン、およびその計算途中には
単精度浮動小数点を使用します。固定小数点化を目的として複雑な処理を追加しません。

波形テーブルの位相は32bit位相アキュムレータで保持します。1サンプルごとに位相増分を加算し、
上位ビットで32要素の波形テーブルを参照します。初期実装では波形サンプル間を補間しません。

各ボイスの左右出力を32bit以上の作業値へ加算し、最終出力時に16bitへ飽和させます。実際の
ミックス作業値を整数と浮動小数点のどちらにするかは、共通実装の計測とRP2350での確認後に
決定します。

## エンベロープとパラメーター更新

エンベロープはサンプルごとに評価しません。512サンプルのPCMブロックを生成する直前に、
各ボイスについて1回だけ評価します。そこで得た音量は、そのブロックの512サンプルすべてへ
一定値として適用します。

同様に、パンから求める左右ゲインもブロック単位で更新します。これにより、サンプル生成の
内側では波形参照、一定ゲインの乗算、ミックス、および位相更新だけを行います。

エンベロープは少なくともAttack、Decay、Sustain、Releaseを持ちます。時間は秒を基準とする
`float`でデータ化し、ブロック経過時間により状態を進めます。短いAttackやDecayはブロック境界へ
量子化されます。これは本システムの意図した挙動とします。

ピッチスイープもブロック単位で評価し、ブロックごとに位相増分を1つ決定します。1ブロック中の
位相増分は一定とします。

## シーケンサ

プロトタイプのBGMデータはPPQN 480、最大8ボイスで構成されています。イントロ後のループと、
ワンショット再生の両方を扱います。

実行時にJSONやMIDIを解釈しません。開発用コンバーターがBGMとSFXのデータを、直接リンク可能な
C++配列へ変換します。イベントは少なくとも次の情報を持ちます。

- 発音時刻
- 音長
- ノート番号
- ベロシティ
- ボイス番号
- 音色番号

イベント時刻、ノート開始、ノート終了、およびSFX発音要求はPCMブロック境界で処理します。
したがって最大約10.67msの時間量子化を許容します。SFX要求キューは固定長とし、容量は実際の
Wizwardの最大要求数を調べてから決めます。

## 初期波形と音色

プロトタイプとの互換性を確認するため、初期実装では次の32サンプル波形を扱います。

- sine
- bell
- square
- pulse
- thin pulse
- saw
- triangle
- spark
- noise

音色は波形番号、ADSR、基本音量、パン、およびピッチスイープのパラメーターを持ちます。
データ構造の具体的な型とサイズは、音源コアの実装とテストを行いながら確定します。
