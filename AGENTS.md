# Pixel Twins 開発規約

Pixel Twinsは、RP2350を対象とするマイコン向けプログラムです。macOS版は共通コードの開発、
検証、デバッグに使用するホスト実装であり、設計の基準をPC側へ移してはいけません。

## メモリと性能

メモリ使用量と実行性能を常に第一に考えてください。

「念のため」という理由だけで、メモリ使用量や実行性能を悪化させてはいけません。
禁止例には以下が含まれます。

- 必要性を示さずにバッファを0クリアする
- 必要性を示さずにトリプルバッファリングを採用する
- 将来使うかもしれないという理由で領域や抽象化を追加する
- 高速な描画経路の内側へ不要な境界判定や条件分岐を追加する

性能とメモリ使用量を天秤にかける必要がある場合は、独断で決めず、選択肢と具体的なコストを
示して相談してください。

実行中の動的メモリ確保と、容量が変化するコンテナの使用は原則として避けます。
固定長アロケーターや、上限が明確な静的領域を優先してください。

## 描画の優先度

描画性能を最優先します。描画に関連するRP2350向け関数は、すべてSRAMへ配置してください。
これにはBG、スプライト、拡大縮小付きスプライト、プリミティブ、Yバケット、フリップ、
LED転送データ生成、およびLED転送処理が含まれます。

共通C++実装を基準とし、SDLの描画機能で別の描画経路を作ってはいけません。SDL実装は、
完成したフレームバッファの表示を中心とする薄いホスト接続層に保ってください。

対象固有のアセンブラ最適化は許容しますが、共通C++実装と同じデータ契約および描画結果を
維持してください。
